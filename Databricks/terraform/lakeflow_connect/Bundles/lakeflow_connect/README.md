# lakeflow_connect — Databricks Asset Bundle

The Databricks workload for the `lakeflow_connect` CDC pipeline. It:

- **builds** the `cdc_bronze_ingest` wheel (vendored at
  `./cdc_bronze_ingest`) via the `artifacts` block, and
- **deploys** one bronze schema + checkpoint volume + one Auto Loader ingestion
  job **per domain**.

## Single source of truth

The per-domain resources (`resources/*.gen.yml`) are **generated by Terraform**
(`Databricks/terraform/lakeflow_connect/ingestion`) from its `domains` map. The
bundle just builds the wheel and deploys whatever generated resource files are
present. UC governance (storage credential, external location, optional
federation) is created by Terraform — bundles dont create those.

## Variables & configuration

### Per-domain job parameters (Terraform-generated)

Each generated job passes these `named_parameters` to the `cdc_bronze_ingest`
wheel. They come from the domain's entry in the ingestion root's
`terraform.tfvars` — **do not hand-edit the `*.gen.yml`**.

| Parameter | Source | Notes |
|-----------|--------|-------|
| `source_path` | `source_infrastructure.bucket` + `prefix` | S3 URI of the CDC root the job reads |
| `source_schema` | `source_schema` | Source DB/schema (lineage + checkpoint namespacing) |
| `target_catalog` | `target_catalog` (default `main`) | UC catalog for the bronze tables |
| `target_schema` | `${resources.schemas.<domain>_schema.name}` | Resolves to the deployed (dev-prefixed) schema name |
| `checkpoint_base` | `/Volumes/<catalog>/<schema>/_checkpoints` | UC managed volume under the schema, built from resource references (omitted → wheel default) |

## Deploy

```bash
# 1. Terraform first — creates UC governance + generates resources/*.gen.yml
cd ../../ingestion
terraform apply

# 2. Deploy the bundle (builds the wheel, creates schemas + jobs)
cd ../bundles/lakeflow_connect
databricks bundle validate
databricks bundle deploy -t dev

# 3. Run a domain's ingestion (e.g. hr)
databricks bundle run hr_cdc_ingest -t dev
```

## Prerequisites

- `DATABRICKS_HOST` / `DATABRICKS_TOKEN` (or a configured CLI profile) for the
  target workspace.
- `uv` on PATH (used by the `artifacts` build).
- The ingestion Terraform applied first (it generates the resource files this
  bundle includes).
